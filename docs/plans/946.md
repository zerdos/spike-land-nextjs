**API Implementation Details:**

1. **GET /api/orbit/[workspaceSlug]/calendar/posts**
   - Query params: `startDate`, `endDate`, `platform`, `status`
   - Returns: Array of `ScheduledPost` with `postAccounts`
   - Enforces workspace access check

2. **POST /api/orbit/[workspaceSlug]/calendar/posts**
   - Body: `{ content, scheduledAt, timezone, platforms[], recurrence?, assets[] }`
   - Checks: `maxScheduledPosts` subscription limit
   - Creates: `ScheduledPost` + `ScheduledPostAccount[]`
   - Returns: Created post with ID

3. **PATCH /api/orbit/[workspaceSlug]/calendar/posts/[postId]**
   - Updates post content, schedule, or platforms
   - Validates: Cannot reschedule already-published posts
   - Returns: Updated post

4. **POST /api/orbit/[workspaceSlug]/calendar/posts/[postId]/publish**
   - Manually triggers publishing
   - Calls social platform APIs
   - Updates `ScheduledPostAccount` status
   - Returns: Publishing results per platform

5. **GET /api/orbit/[workspaceSlug]/calendar/recommendations**
   - Returns: AI-generated `CalendarContentSuggestion[]`
   - Uses: Brand Brain for voice consistency
   - Analyzes: Historical engagement, content gaps
   - Returns: Suggestions with confidence scores

6. **GET /api/orbit/[workspaceSlug]/calendar/best-times**
   - Returns: `PostingTimeRecommendation[]` per account
   - Sources: Historical data OR industry benchmarks
   - Groups by: Platform and day of week

7. **GET /api/orbit/[workspaceSlug]/calendar/gaps**
   - Analyzes: Scheduled posts vs optimal times
   - Returns: Array of gap windows with suggested platforms
   - Highlights: High-engagement missed opportunities

8. **POST /api/orbit/[workspaceSlug]/calendar/generate-content**
   - Body: `{ platform, suggestedFor, keywords? }`
   - Uses: Brand Brain + AI content generation
   - Consumes: `usedAiCredits` (check `monthlyAiCredits` limit)
   - Returns: Generated content with variations


**Cron Job Logic:**

```typescript
// src/app/api/cron/publish-scheduled-posts/route.ts
export async function GET(request: Request) {
  // 1. Find posts due for publishing (scheduledAt <= now, status = SCHEDULED)
  // 2. For each post:
  //    - Get all ScheduledPostAccount entries
  //    - For each account:
  //      - Check retry count < maxRetries
  //      - Attempt to publish via SocialPostPublisher
  //      - Update ScheduledPostAccount with result
  //      - If success: set status=PUBLISHED, save platformPostId
  //      - If failure: increment retryCount, save errorMessage
  // 3. Handle recurring posts:
  //    - Calculate nextOccurrenceAt from recurrenceRule
  //    - Create new ScheduledPost for next occurrence
  // 4. Return summary of published/failed posts
}
```

**Retry Logic:**

- Max retries: 3 (from `ScheduledPost.maxRetries`)
- Exponential backoff: 5min, 15min, 30min
- After max retries: Set status to `FAILED`

### 9. Recurring Posts

**Implementation Strategy:**

Use RRULE format (RFC 5545) for recurrence:

```typescript
// Example: Weekly on Monday at 9am
recurrenceRule: "FREQ=WEEKLY;BYDAY=MO;UNTIL=20261231T090000Z";

// Parsing and next occurrence
import { RRule } from "rrule";

const rule = RRule.fromString(post.recurrenceRule);
const nextOccurrence = rule.after(new Date());
```

**Cron Job Handling:**

After publishing a recurring post:

1. Parse `recurrenceRule` with `rrule`
2. Calculate `nextOccurrenceAt`
3. Create new `ScheduledPost` with same content
4. Link to original post for history
5. Update original with `nextOccurrenceAt`
